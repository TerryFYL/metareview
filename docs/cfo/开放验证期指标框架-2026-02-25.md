## 0. 执行摘要

MetaReview 当前运营成本约 **$0.83/月** (仅域名分摊)，Cloudflare 免费层足以支撑 **日活 500+ 用户**。产品在技术上已达到发布标准 (188/188 交叉验证通过)，但 **$0 收入、~2 用户** 意味着商业验证尚未开始。

核心结论:
1. 30 天内追求收入不现实也不明智 -- 北极星指标应该是 **完成分析的用户数 (Completed Analyses)**
2. Kill Date 的判断标准不是"有没有赚钱"，而是"有没有人用、用得深不深"
3. 变现模式建议 **Freemium + 机构订阅**，但引入付费的前提是至少 50 个活跃用户
4. 当前成本结构几乎为零，这是战略优势 -- 不需要盈利压力就能耐心验证

## 1. Go/No-Go 指标框架 (Kill Date 4/25 决策矩阵)

### 设计原则

30 天开放验证期，从 0 用户起步，不能用收入来判断。用 **领先指标 (Leading Indicators)** 预测未来收入潜力。

### 决策矩阵

| # | 指标 | 定义 | GO 阈值 | PIVOT 阈值 | 数据源 |
|---|------|------|---------|------------|--------|
| 1 | **完成分析数 (Completed Analyses)** | 用户完成 >= 3 个研究的 Meta 分析次数 | >= 10 次 | < 3 次 | `export_*` 事件 |
| 2 | **周回访率 (Weekly Return Rate)** | 第 2 周仍有活跃的用户占第 1 周用户比例 | >= 30% | < 10% | `page_view` 日数据 |
| 3 | **独立用户数 (Unique Visitors)** | 30 天内不重复访客 (以日 page_view 峰值估算) | >= 200 | < 30 | `page_view` 事件 |
| 4 | **深度使用率 (Depth Rate)** | 使用 >= 3 个核心功能的用户占比 | >= 20% | < 5% | `tab_switch` 多步事件 |
| 5 | **邮件订阅数 (Email Subscribers)** | 主动留下邮箱的用户数 | >= 30 | < 5 | `EMAILS` KV count |

### 决策规则

```
IF 指标 1 >= 10 AND 指标 3 >= 200 AND 至少 3/5 指标达到 GO:
    --> GO: 继续投入，进入增长期

IF 指标 1 在 3-9 AND 指标 3 在 30-199:
    --> ITERATE: 不 pivot 产品，但调整分发策略，延长验证 30 天

IF 指标 1 < 3 OR 指标 3 < 30:
    --> PIVOT: 产品假设可能错误，重新评估方向
```

### 为什么是这 5 个指标

1. **完成分析数** -- 这是产品核心价值的直接度量。如果没人用它做完一个 Meta 分析，产品价值为零。10 次意味着至少有 5-10 个真实研究者在用。
2. **周回访率** -- Meta 分析不是一次性任务，一个项目通常跨 2-4 周。回访率 < 10% 说明用户试了一下就走了，产品没有粘性。
3. **独立用户数** -- 200 是基本的统计显著性门槛。低于 30 个用户，所有其他指标都没有统计意义。
4. **深度使用率** -- 区分"看了一眼"和"真正在用"。核心功能包括: 文献检索、数据录入、统计分析、报告导出。
5. **邮件订阅数** -- 愿意留邮箱 = 愿意被再次触达 = 有真实兴趣。这是未来变现的漏斗入口。

### 指标采集现状评估

| 指标 | 当前可追踪? | 缺口 |
|------|-------------|------|
| 完成分析数 | 部分 -- `export_*` 事件已追踪 | 需要确认事件触发在"真正导出"而非"点击按钮" |
| 周回访率 | 否 -- 无用户身份标识 | KV analytics 只记事件计数，不区分用户。**需要添加匿名 session 指纹** |
| 独立用户数 | 否 -- 同上 | 可通过每日 page_view 峰值粗略估算，或添加匿名日活统计 |
| 深度使用率 | 部分 -- `tab_switch` 已追踪 | 需要 session 级别追踪才能计算"同一用户用了几个功能" |
| 邮件订阅数 | **是** | `EMAILS` KV 已有 `_count` |

**关键缺口: 当前 analytics 缺乏用户级追踪。** 建议在开放验证前添加基于 localStorage 的匿名 visitor ID (非 cookie，符合隐私要求)，使得可以计算 DAU、回访率、漏斗转化。

### 2.2 Cloudflare 免费层容量分析

```
当前 dist 大小: ~4.3 MB (含 PDF worker)
首次加载: ~2.5 MB (gzip 后 ~800 KB)
后续访问: 0 (浏览器缓存)

Cloudflare Pages 免费层:
- 带宽: 无限
- 请求: 无限
- 构建: 500/月 (足够)

Cloudflare KV 免费层 (每日上限):
- 100,000 reads/天
- 1,000 writes/天 <-- 瓶颈

每个 page_view 事件消耗:
- 2 reads (获取当前计数 + 获取汇总)
- 2 writes (写入计数 + 写入汇总)
- 如果有 referrer: +1 read +1 write

KV write 瓶颈计算:
- 1,000 writes/天
- 每个 page_view ~2-3 writes
- 每个 tab_switch ~2 writes
- 每个功能事件 ~2 writes
- 假设平均用户触发 10 个事件 = ~20 writes
- 日活上限: 1,000 / 20 = ~50 DAU (仅 analytics 限制)

Workers AI 免费层 (每日上限):
- 10,000 neurons/天
- Llama 3.1 8B: ~50-100 neurons/请求 (取决于 token 数)
- 预估: 100-200 次 AI 筛选/天
- 日活上限: 如果平均每人筛 20 篇 = ~5-10 个并行用户
```

**关键发现: KV writes 是第一个会撞到的天花板。** 50 DAU 时 analytics 写入可能超限。Workers AI 在 5-10 个同时使用 AI 筛选的用户时会撞墙。

### 2.4 变现路径分析

学术工具的变现模式，按可行性排序:

#### 路径 A: Freemium (推荐)

```
免费层 (永久):
  - 单个 Meta 分析项目
  - 最多 30 个研究
  - 基础统计 (固定/随机效应)
  - 森林图
  - 基础报告导出 (Markdown)

Pro ($9-19/月 或 $89-179/年):
  - 无限项目
  - 无限研究数
  - 高级统计 (亚组、敏感性、累积、网络)
  - AI 文献筛选 (配额更高)
  - AI 数据提取
  - DOCX/HTML 报告导出
  - 分享链接
  - 数据导入导出 (CSV/RIS)

机构版 ($499-1999/年):
  - 团队协作 (未来功能)
  - 管理后台
  - 优先支持
  - 培训/入职
  - SLA
```

竞品对标:
- CMA (Comprehensive Meta-Analysis): $295/年 (学术版)
- Covidence: $240/实例 (机构常见)
- Rayyan: 免费基础版 + 付费 AI 功能
- RevMan: 免费 (Cochrane 资助)

**定价锚点建议: $9-19/月 (个人研究者)，显著低于 CMA 的 $295/年。**

#### 路径 B: 按量计费 (AI 功能)

```
免费: 50 次 AI 筛选/月 + 10 次 AI 提取/月
付费: $0.05/次 AI 筛选, $0.10/次 AI 提取
或积分包: $5 = 200 AI 筛选 + 50 AI 提取
```

优点: 对轻度用户友好，成本和收入线性相关
缺点: 收入不可预测，用户有"花钱焦虑"

#### 路径 C: 一次性付费 (不推荐)

```
$29-49 终身使用
```

缺点: 无持续收入，无法支撑持续开发。学术工具用户付费意愿低且习惯周期性使用。

#### 推荐: 路径 A (Freemium) 为主 + 路径 B 作为升级诱因

**价值指标 (Value Metric) 选择: "项目数 + 研究数上限"**

理由:
1. 与用户获得的价值线性相关 -- 做更多 Meta 分析 = 更多论文产出 = 更大价值
2. 新用户可以免费体验完整流程 (1 个项目 + 30 个研究 = 足够做一篇论文)
3. 重度用户 (多个项目并行) 自然转入付费
4. AI 功能作为额外价值加速器，不作为核心限制

## 3. 免费 -> 付费转换节点

### 3.1 过早收费的风险

```
当前状态: 0 用户, 0 社区, 0 口碑
学术工具的获客特点:
  - 口碑驱动 (实验室内传播、论文方法部分提及)
  - 决策周期长 (研究者先试用，下一个项目才决定用)
  - 价格敏感 (PhD 学生/博士后个人无预算)
  - 机构采购慢 (大学采购流程 3-6 个月)
```

**结论: 在获得 50 个活跃用户之前引入付费墙 = 自杀。**

原因:
1. 没有用户就没有口碑，没有口碑获客成本无限高
2. 学术用户第一反应是找免费替代品 (R metafor 是免费的)
3. 竞品 RevMan 免费，Rayyan 基础版免费
4. 一个 PhD 学生用了你的工具发了论文，会推荐给整个实验室 -- 这个口碑价值远超 $19/月

### 3.2 转换触发器 (按阶段)

| 阶段 | 条件 | 动作 |
|------|------|------|
| **阶段 0: 当前** | < 50 用户 | 完全免费，所有功能开放。专注获取用户和反馈 |
| **阶段 1: 初步验证** | >= 50 DAU 持续 2 周 | 引入"Pro"标签 (视觉区分但不限制功能)。收集定价意向: "这个功能值多少钱?" |
| **阶段 2: 软付费** | >= 200 DAU + >= 30 邮件订阅 | 对**新用户**引入 Freemium 限制 (老用户祖父条款保护)。免费层: 1 项目 + 30 研究 |
| **阶段 3: 正式变现** | >= 500 DAU + >= 5 次付费意向表达 | 上线支付。Pro $9/月 或 $89/年。机构联系邮箱 |
| **阶段 4: 扩展** | >= 50 付费用户 | 考虑团队功能、机构版。调研机构采购流程 |

### 3.3 定价验证方法 (Campbell 方法论)

不要问用户"你愿意付多少钱" -- 他们会说"免费最好"。用这些方法:

**Van Westendorp 价格敏感度测试 (在阶段 1 执行):**

在应用内嵌入 4 个问题 (随机对 10% 的活跃用户展示):
1. 你觉得这个工具**便宜到不可信**的价格是? ($/月)
2. 你觉得这个工具**很划算**的价格是? ($/月)
3. 你觉得这个工具**有点贵但能接受**的价格是? ($/月)
4. 你觉得这个工具**太贵了**的价格是? ($/月)

交叉点 = 最优定价区间。

## 4. 30 天北极星指标

### 4.1 北极星: 完成的分析数 (Completed Analyses)

**定义**: 用户导出 (export_md, export_docx, export_html, export_json) 且该分析包含 >= 3 个研究的次数。

**为什么是这个指标:**

```
收入 = 用户数 x 转化率 x ARPU

在 30 天内:
- 用户数: 可以驱动 (分发策略)
- 转化率: 还不能测 (还没收费)
- ARPU: 还不能测 (还没定价)

但完成的分析数是收入的最强领先指标:
- 它证明用户从产品获得了真正的价值 (完成了一个分析)
- 完成分析的用户有最高的付费意愿
- 完成分析的用户会在论文中提到工具 -> 免费获客
```

### 4.2 辅助追踪仪表盘

```
每日追踪:
  page_views          -- 流量 (做大漏斗顶部)
  unique_visitors*    -- 去重后的真实用户 (*需要添加匿名 ID)
  tab_switch(>=3)     -- 深度使用 (用户走了 >= 3 步)
  export_*            -- 完成分析
  email_subscribe     -- 留存潜力
  al_screening_start  -- AI 功能使用 (未来付费功能)
  pdf_upload          -- 高价值行为

每周追踪:
  new vs returning    -- 新用户与回访用户 (*需要匿名 ID)
  漏斗: page_view -> tab_switch -> export -- 转化漏斗
  referrer sources    -- 哪个渠道带来的用户质量最高

核心公式:
  完成的分析数 = page_views x 深度使用率 x 完成率

  假设验证成功的数字:
  1000 page_views x 20% 深度 x 10% 完成率 = 20 analyses/月
  这需要 ~33 page_views/天
```

### 4.3 目标拆解

```
30 天目标: >= 10 次完成的分析

倒推:
  完成率假设 10% (使用 3+ 功能的用户中)
  -> 需要 100 个深度用户 (使用 >= 3 功能)

  深度率假设 20% (所有访客中)
  -> 需要 500 个独立访客

  -> 日均 ~17 个独立访客

  获客渠道 (无预算):
  - 学术社区帖子 (r/AcademicPsychology, r/biostatistics, ResearchGate)
  - 系统综述相关论坛
  - 中文学术圈 (知乎, 小红书, 微信公众号)
  - 产品社区 (Product Hunt, Hacker News)
  - SEO (meta-analysis tool, free meta-analysis software)

  现实预期:
  - 1 个好帖子 = 50-200 visits/天 (峰值), 5-20/天 (长尾)
  - SEO: 0 (30 天内不会有显著搜索流量)
  - 总计: 需要至少 3-5 个有效的社区分发动作
```

| 10% 完成率 | **估算** | 学术工具偏保守估计 |

### 关键风险

1. **无法获取用户 (分发失败)** -- 概率高。免费工具不缺，用户发现成本高。对策: 集中精力做 3-5 个高质量社区分发。
2. **用户来了但不完成分析** -- 产品复杂度高，中途放弃。对策: 引导流程 (已有) + 降低完成门槛。
3. **AI 配额耗尽影响体验** -- 多用户同时使用 AI 筛选。对策: 已有降级逻辑，确保不崩溃只是降级。
4. **竞品免费 (RevMan/R)** -- 付费转化率可能极低。对策: 定位差异化 (无需安装/无需 R 编程/AI 辅助)。
